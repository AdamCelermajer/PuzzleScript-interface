Pseudocode

Main Runtime Loop

client/runtime/runner.py
Main loop Runtime <-> engine modules <-> ARC/PuzzleScript engine
  1. frame ← env.reset() ARC/PuzzleScript starts the game
  2. state ← perceiver.perceive(frame)
  3. memory.append_state(state)
  4. while step count < max_steps do
  5. decision ← planner.choose_action()
  6. P ← rulebook.predict(state, decision.action)
  7. next_frame ← env.step(decision.action) runtime sends the action to the ARC/PuzzleScript engine
  8. next_state ← perceiver.perceive(next_frame)
  9. rulebook.record_prediction_result(state, action, next_state, P)
  10. memory.append_action_result(action, next_state)
  11. unexplained ← (P is empty) or (next_state not in P)
  12. if unexplained then planner.clear_llm_plan()
  13. if decision is exploratory or unexplained then inducer.propose_from_memory(memory)
  14. if board is stuck, won, or game-over then handle reset/stop
  15. state ← next_state

Entrypoint

client/run_arc_agent.py
File role Wire one runnable session
  1. cfg ← Config(game id, backend URL, max steps)
  2. dashboard ← ScreenDashboard(game id)
  3. env ← ArcadeEnv(game id, backend URL, dashboard renderer)
  4. llm ← LlmClient(cfg, dashboard event sink)
  5. engine ← EngineArchitecture.from_config(cfg, llm)
  6. loop ← RuleReasoningLoop(env, perceiver, memory, rulebook, planner, inducer)
  7. dashboard.run_engine(loop.run)

Engine Wiring

client/engine/architecture.py
File role Build the reasoning modules
  1. base_path ← rules directory for the current game
  2. memory ← EngineMemory(base_path / timeline.jsonl)
  3. rulebook ← Rulebook(base_path)
  4. verifier ← RuleVerifier(memory)
  5. inducer ← RuleInducer(llm, rulebook, verifier)
  6. planner ← Planner(rulebook, memory, llm)
  7. return EngineArchitecture(perceiver, memory, rulebook, inducer, planner)

Planner

client/engine/planner.py
File role Choose the next action
  1. s ← memory.current_state()
  2. A ← available non-reset actions from s
  3. if active plan has a next action then
  4. return PlanDecision(next action, reason = active_plan)
  5. if no game goal exists then
  6. game goal ← goal_manager.ensure_goal(memory)
  7. plan ← BreadthFirstRuleSearch(s, A, rulebook)
  8. if plan reaches a win state then
  9. store plan as the active plan
  10. return PlanDecision(first action in plan, reason = rule_plan)
  11. context ← latest transition text and images from memory
  12. (subgoal, probe) ← llm.ask_for_subgoal_action(s, A, known rule summaries, context)
  13. store probe as the active plan for subgoal
  14. return PlanDecision(first action in probe, reason = explore_subgoal, exploratory = true)

Rule Learning

client/engine/induction.py + client/engine/verifier.py
File role Turn fresh transitions into accepted executable rules
  1. E ← memory.recent_transitions(1)
  2. if E is empty then return
  3. (events, images) ← memory.latest_visual_context()
  4. K ← rulebook.known_rules_text() accepted summaries only
  5. C ← llm.call_json(rule prompt(events, K), images)
  6. for all candidate rules c in parse(C) do
  7. failures ← empty list
  8. for all evidence id e cited by c do
  9. t ← memory.transition_by_id(e)
  10. if t is missing or c.action != t.action then record failure
  11. else if t.after not in c.predict(t.before) then record failure
  12. if failures is empty then rulebook.add_generalized_rule(c)
  13. else log rejected candidate and continue
