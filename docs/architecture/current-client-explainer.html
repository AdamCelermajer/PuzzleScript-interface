How the current LLM client actually works

A code-grounded map of the current agent: what starts it, what owns each decision, where the LLM is called, and how observations become stored rules.

Entry point client/run_arc_agent.py builds config, env, LLM, dashboard, architecture, and loop.
Truth source The environment is always stepped for real. Rules are judged against observed transitions.
LLM role today It proposes exploratory plans and generalized rule candidates. It does not own the whole loop.
Modularity Module boundaries now match the diagram; agent.py remains only as a compatibility shim.

Small Abstract Flow

This is the useful level of abstraction: one loop, one environment, one rule memory, and occasional LLM calls.

1

Start CLI

Parse game, backend, mode, max steps.

2

Build runtime

Create environment, LLM client, dashboard, and engine modules.

3

Reset

Ask ARC surface for the first frame.

4

Perceive

Convert raw frame data into stable engine state.

5

Choose action

Planner uses verified rules first, LLM fallback second.

6

Act

Step the real environment with the chosen action.

7

Record truth

Store before/action/after evidence and exact transition memory.

8

Maybe induce

Ask LLM for generalized rules, then verify them against evidence.

EngineRulebook.predict(state, action):
    predictions = verified_generalized_rules(state, action)
    if predictions:
        return predictions

    return exact_transition_predictions(state, action)

EngineRulebook.record_prediction_result(...):
    if actual_after in predictions:
        mark matching rule hit
    else:
        reject failing prediction rules

EngineRulebook.record_observed_transition(record):
    library.record_transition(record)

RuleFirstPlanner.choose_action(state, frame):
    actions = available_actions(frame)

    plan = plan_to_win_using_rulebook(state, actions)
    if plan:
        clear_pending_llm_plan()
        return verified plan action

    if pending LLM plan:
        return next LLM action

    response = ask_llm_for_subgoal(
        board=state.rows(),
        actions=actions,
        known_rules=rulebook.known_rules_text(),
        recent_events=memory.recent(5),
    )
    store response.plan[1:] as pending
    return response.plan[0]

RuleInducer.propose_from_recent(game, records):
    events = format records as before/action/after text
    data = llm.call_json(rule_prompt(events))
    candidates = parse GeneralizedRule objects

    for candidate in candidates:
        verified = verifier.verify(candidate)
        library.add_generalized_rule(verified)

RuleVerifier.verify(rule):
    for evidence_id in rule.evidence_ids:
        record = find transition by id
        if record.after not in rule.predict(record.before):
            return rejected rule
    return verified rule

RuleReasoningLoop.run_learning():
    frame = env.reset()
    state = perceiver.perceive(frame)

    while steps < max_steps:
        decision = planner.choose_action(state, frame)
        predictions = rulebook.predict(state, decision.action)
        outcome = actions.execute(frame, state, decision)

        rulebook.record_prediction_result(outcome, predictions)
        record = memory.record_transition(...)
        rulebook.record_observed_transition(record)

        if prediction diverged or checkpoint:
            inducer.propose_from_recent(game_id, memory.recent(5))

        frame = outcome.after_frame
        state = outcome.after_state
